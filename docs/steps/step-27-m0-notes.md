detached from the workspace via an empty `[workspace]` table). The scratch crate has two `#[data]`

build — the step-10 `generate swift` precedent; ~15 s).

Precondition checked before generation: `boltffi_cli v0.28.0`, no `?rev=` (registry build). The
probe re-asserts this itself.

**Result:** both items appear in the generated Kotlin as real bindings:

```
| #[data] item                               | in Kotlin |
| AlwaysHere (unconditional control)         | present   |
| IosOnlyHint (#[cfg(target_os="ios")])      | present   |  ← data class IosOnlyHint(...) + WireReader decoder
```

So BoltFFI's bindgen evaluates no `#[cfg]`: a gated item joins every target's binding surface (the
union), `generate` exits 0, and the missing symbol would only surface at link/run time. That is the
**benign** failure direction for the merge — and because we took the contract-side exit (declare the
surface uniform, Q10), no gated item exists to leak. Verdict recorded in
`upstream/boltffi/08-bindgen-ignores-cfg-attributes.md` (framing changed from "no runtime probe yet"
to "now runtime-probed", with the artifact pointer and what it showed).

## Decisions taken (smallest-reversible)

- **The C3 `Capability::PriorityHint` column is removed entirely, not kept as always-absent.** With
  the trait deleted, `AdapterFactory::priority_hint()` can no longer type-check, and priority is now
  *uniform* (not divergent), so it does not belong in a divergence matrix. Reversible: re-adding a
  capability variant is a localized change if priority ever regains a real per-engine seam. (The
  step doc explicitly listed `conformance/c3.rs` among the references to fix, so this is in scope,
  not a §9/ARCHITECTURE decision.)
- **Neutral internal names `NativeAdapter` / `NativeFactory`** for the merged shim (was
  `SwiftAdapter`/`KotlinAdapter`, `SwiftFactory`/`KotlinFactory`). Private, non-FFI, freely
  renamable; "native" reads correctly for both URLSession and OkHttp.
- **Dual-platform prose in the merged doc comments**, keeping both former files' substance
  (URLSession + OkHttp specifics named side by side) rather than dropping either.
